# OpenClaw Manager 用户使用说明书

**版本**：v1.0.0
**日期**：2026-03-28
**适用范围**：最终用户

---

## 一、产品简介

OpenClaw Manager 是 OpenClaw 的图形化管理工具，让您无需命令行即可完成 OpenClaw 的配置、管理和监控。

### 核心功能

| 功能 | 说明 |
|------|------|
| 环境检测 | 自动检测 Node.js、OpenClaw、Gateway 状态 |
| 模型配置 | 添加 AI 模型提供商，管理模型，为 Agent 选择模型 |
| 渠道管理 | 启用/配置 24 个消息渠道（WhatsApp、Telegram、飞书等） |
| Agent 管理 | 创建和管理 AI Agent |
| 健康监控 | 自动诊断系统状态，一键修复问题 |
| 备份恢复 | 加密备份配置，支持恢复 |
| 配置编辑 | 可视化编辑 OpenClaw 配置文件 |

## 二、安装与启动

### 系统要求

- Windows 10 或 Windows 11
- Node.js >= 22.14（推荐 24.x）
- OpenClaw 已安装（`npm install -g openclaw@latest`）

### 安装步骤

1. 下载 `openclaw-manager.exe` 安装包
2. 双击运行安装程序
3. 按向导完成安装
4. 桌面出现 "Hi OpenClaw" 图标

### 首次启动

首次打开应用会显示环境检测引导：

1. **检测 Node.js**：确认已安装且版本达标
2. **检测 OpenClaw**：确认已安装
3. **检测 Gateway**：确认 Gateway 运行中（未运行可点击"启动"）

全部检测通过后，点击"开始使用"进入主界面。

## 三、功能使用说明

### 3.1 Dashboard（仪表盘）

显示系统运行状态概览：
- Gateway 运行状态
- Agent 数量、渠道数量
- Node.js / OpenClaw 版本信息

### 3.2 Models（模型配置）

**查看已有配置**：
- 切换到 "Provider 列表" 页签
- 展开各 Provider 查看已配置的模型
- 内置 Provider（如 OpenRouter）显示紫色"内置"标签

**添加新 Provider**：
1. 点击 "添加 Provider"
2. 填写 Provider ID、显示名称、Base URL、API Key
3. 点击"保存"

**添加模型**：
1. 展开目标 Provider
2. 点击 "添加模型"
3. 填写模型 ID、显示名称、上下文窗口、最大输出 Token
4. 点击"添加"

**配置 Agent 模型**：
1. 切换到 "Agent 模型配置" 页签
2. 为"全局默认"和每个 Agent 选择主模型和 Fallback 模型
3. 点击"保存"

> **重要**：所有修改会直接写入配置文件，立即生效。

### 3.3 Channels（渠道配置）

- 查看所有 24 个消息渠道
- 点击渠道卡片可启用/禁用
- 启用后需配置相应凭证（如飞书的 App ID/Secret）

### 3.4 Agents（Agent 管理）

**创建 Agent**：
1. 点击 "创建 Agent"
2. 第 1 步：填写名称
3. 第 2 步：选择平台（飞书/Telegram）
4. 第 3 步：配置凭证（App ID、App Secret 等）
5. 第 4 步：确认创建

**删除 Agent**：点击 Agent 卡片上的删除按钮

### 3.5 Skills（Skill 市场）

- "本地已安装"页签：查看每个 Agent 已安装的 Skill
- 可以删除不需要的 Skill

### 3.6 Health（健康监控）

- 自动每 60 秒刷新诊断结果
- 绿色标签 = 正常，红色标签 = 异常
- 异常项可点击"修复"按钮一键修复
- 底部可查看 Gateway 日志

### 3.7 Backup（备份恢复）

**创建备份**：
1. 点击 "创建备份"
2. 选择备份内容（配置/Agent/凭证）
3. 设置加密密码
4. 点击"开始备份"

**恢复备份**：
1. 在备份列表中找到目标备份
2. 点击"恢复"
3. 输入备份时设置的密码
4. 确认恢复

### 3.8 Config（配置编辑）

- "配置段"页签：按类别查看配置
- "原始 JSON"页签：直接编辑 JSON（慎用）
- "备份"页签：查看配置文件的自动备份

### 3.9 Settings（设置）

- **Gateway 设置**：端口、Tailscale 模式
- **安全设置**：私聊策略、Gateway 认证方式
- **界面设置**：主题（浅色/深色/跟随系统）
- **其他设置**：自动更新、遥测数据
- **关于**：版本信息、项目链接

## 四、常见问题

### Q1：Gateway 显示"未运行"怎么办？
点击 Header 右上角的 Gateway 状态标签，或到 Health 页面点击"启动 Gateway"。

### Q2：暗色主题切换后没有反应？
当前版本切换主题后会立即生效。如果看到部分页面颜色异常，请切换到浅色主题再切回来。

### Q3：备份文件在哪里？
备份文件保存在 `~/.openclaw/backups/` 目录下，文件扩展名为 `.ocbak`。

### Q4：如何重新触发首次引导？
删除浏览器 localStorage 中的 `ocm-onboarding-completed` 键值，然后刷新页面。

### Q5：Chat 页面为什么跳转到浏览器？
OpenClaw Gateway 使用复杂的 WebSocket 协议，OpenClaw Manager 专注于配置和管理，聊天功能通过官方 Control UI 提供。

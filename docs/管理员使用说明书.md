## 三、部署与维护

### 3.1 从源码构建

```bash
# 前置要求
# - Node.js >= 22.14
# - Rust 1.70+ (MSVC Build Tools 2022)
# - pnpm

git clone https://github.com/husir124/openclaw-manager.git
cd openclaw-manager
pnpm install
pnpm tauri build
```

产物位于 `src-tauri/target/release/openclaw-manager.exe`

### 3.2 开发模式

```bash
pnpm tauri dev
```

### 3.3 代码质量检查

```bash
# TypeScript 编译
npx tsc --noEmit --strict

# Rust Clippy
cd src-tauri && cargo clippy

# 单元测试
pnpm vitest run
```

### 3.4 配置文件安全

**重要**：Manager 只读写 `openclaw.json`，不会修改 OpenClaw 配置的结构。遵循以下规则：

1. **不添加 OpenClaw 不认识的字段**：`meta` 对象只允许 `lastTouchedVersion` 和 `lastTouchedAt`
2. **不修改 OpenClaw 的内部文件**：不操作 `sessions/`、`agents/` 等目录
3. **备份后再修改**：每次写入配置前先创建备份

### 3.5 性能监控

| 指标 | 目标 | 实际 |
|------|------|------|
| 冷启动 | < 3s | ~1-2s |
| 页面切换 | < 200ms | < 100ms |
| 内存占用 | < 200MB | ~80-120MB |
| 安装包 | < 30MB | 12.81MB |

## 四、安全说明

### 4.1 敏感数据处理

| 数据 | 存储方式 | 保护措施 |
|------|----------|----------|
| API Key | openclaw.json（明文） | 前端 UI 遮蔽显示 |
| Gateway Token | 文件系统 | 本地回环访问 |
| 备份文件 | AES-256-GCM 加密 | 用户密码保护 |

### 4.2 网络安全

- 本地 WebSocket：`ws://127.0.0.1:18789`（仅本地回环）
- 不对外暴露端口
- Tauri IPC 限制前端只能调用已注册的命令

### 4.3 文件权限

- 配置文件：用户目录下，操作系统级权限保护
- 备份文件：加密存储，无密码无法解密
- 日志文件：自动脱敏，不记录 API Key

## 五、故障排查

### 5.1 应用无法启动

1. 检查 Windows 版本（需 Windows 10+）
2. 检查 WebView2 运行时是否安装
3. 查看 `%LOCALAPPDATA%\openclaw-manager\logs\` 下的日志

### 5.2 Gateway 连接失败

1. 确认 Gateway 正在运行：`openclaw gateway status`
2. 确认端口未被占用：`netstat -ano | findstr 18789`
3. 检查防火墙设置

### 5.3 配置文件损坏

1. 从 `~/.openclaw/backups/` 恢复最近的备份
2. 或运行 `openclaw doctor --fix` 修复

### 5.4 主题切换无效

1. 清除浏览器缓存（Tauri WebView 缓存）
2. 删除 `%LOCALAPPDATA%\openclaw-manager\` 下的缓存目录
3. 重启应用
